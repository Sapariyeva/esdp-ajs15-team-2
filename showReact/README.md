## Базовый URL
Все эндпоинты относятся к базовому URL: "http://localhost:8000/"

## Эндпоинты


#### POST-запрос на  '/artists'
- Описание: создать нового артиста
- Параметры:
    - name (строка) - имя или никнейм артиста
    - photo (строка) - фотография артиста
    - id (число) -  ID артиста
    - description (строка) - информация об артисте
- Ответ, если все обязательные поля заполненные: 
    - Код состояния: **200  OK** 
- Тело ответа:
``` 
json:
{
  "id": "1",
  "name": "Drake",
  "photo": "Drake_photo.jpg",
  "description": "Some description"
}  
```
- Ответ, если все обязательные поля незаполненные: 
  - Код состояния: **400 Bad Request** 
  - Тело ответа: 
``` 
json:
{
  "error": "Name and photo must be present in the request"
}  
```

  - Код состояния: **400 Bad Request**


#### Инструкция по запуску фикстур
Нужно провалиться в серверную часть с помощью терминала и запустить команду "npm run seed". При запуске данного скрипта, программа заполняет все сущности в базе данных. 

#### Аутентификация
- При регистрации пользователь должен заполнить username и password. Эти данные отправляются в базу данных и хранятся там. Пароль сохраняется в зашифрованном виде, для безопасности.
- При авторизации пользователь так же заполняет поля username и password. Если данные сходятся с теми, что есть в базе данных, то ему выдается токен и он хранится в localstorage браузера и в базе данных. Наличие токена означает, что пользователь прошел авторизацию и ему дается возможность просмотреть треки определенного альбома и свою историю прослушивания. 
- При разлогированию(logout) данный токен удаляется из localstorage и базы данных. Ему становятся не доступны функции для авторизованного пользователя и что бы их получить вновь, юзеру нужно пройти повторную авторизацию
